# Security Policy

## Supported versions

This repository tracks the current main branch.

## Reporting a vulnerability

If you find a security issue, report it privately if the platform supports GitHub Security Advisories for this repository.

If private reporting is not available, open an issue with:

- a short summary
- the affected file or flow
- reproduction steps
- impact description

Do not include secrets, credentials, or live tokens in the report.

## What not to report publicly

- API keys
- access tokens
- passwords
- private keys
- session cookies
